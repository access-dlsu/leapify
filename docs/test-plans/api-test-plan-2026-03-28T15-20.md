## Authentication Requirements

| Role | Description | How to Simulate in Tests |
|------|-------------|--------------------------|
| guest | No Authorization header | Omit the header entirely |
| user | Authenticated student | Provide a mocked valid Firebase JWT; the auth middleware must accept it |
| admin | Authenticated admin | Provide a mocked Firebase JWT with `admin: true` custom claim |

| Binding | Type | Required By | Notes |
|---------|------|-------------|-------|

### Events

#### API-EVENTS-001: List published events
- **Source:** `src/routes/events.ts`
- **Endpoint:** `GET /events`
- **Priority:** P0
- **Auth Required:** guest
- **Preconditions:**
  - 1 event with status `published` exists.
  - 1 event with status `draft` exists.
- **Request:**
  - Method: GET
  - Path: `/events`
- **Expected Response:**
  - Status: 200
  - Body shape: `{ data: [...] }` containing ONLY the `published` event.
- **Side Effects to Assert:**
  - Cache-Control header should be present with `max-age=604800`.
  - ETag header should be present.

#### API-EVENTS-002: List published events with ETag matching
- **Source:** `src/routes/events.ts`
- **Endpoint:** `GET /events`
- **Priority:** P1
- **Auth Required:** guest
- **Preconditions:**
  - KV contains the `events:etag` from a previous run.
- **Request:**
  - Method: GET
  - Path: `/events`
  - Headers: `If-None-Match: <the-etag>`
- **Expected Response:**
  - Status: 304

#### API-EVENTS-004: Get published event by slug
- **Source:** `src/routes/events.ts`
- **Endpoint:** `GET /events/:slug`
- **Priority:** P0
- **Auth Required:** guest
- **Preconditions:**
  - An event with slug `leap-2025` and status `published` exists.
- **Request:**
  - Method: GET
  - Path: `/events/leap-2025`
- **Expected Response:**
  - Status: 200
  - Body shape: `{ data: { slug: 'leap-2025', ... } }`

#### API-EVENTS-006: Get event slots info
- **Source:** `src/routes/events.ts`
- **Endpoint:** `GET /events/:slug/slots`
- **Priority:** P0
- **Auth Required:** guest
- **Preconditions:**
  - An event with slug `leap-2025` exists.
- **Request:**
  - Method: GET
  - Path: `/events/leap-2025/slots`
- **Expected Response:**
  - Status: 200
  - Body shape: `{ data: { total: number, available: number, ... } }`
  - Cache-Control header should be `public, max-age=5, stale-while-revalidate=5`.

#### API-EVENTS-008: Create event with admin token
- **Source:** `src/routes/events.ts`
- **Endpoint:** `POST /events`
- **Priority:** P0
- **Auth Required:** admin
- **Preconditions:** None
- **Request:**
  - Method: POST
  - Path: `/events`
  - Headers: `Authorization: Bearer <admin-token>`
  - Body: `{ "slug": "new-event", "categoryName": "Cat", "categoryPath": "/cat", "title": "New Event" }`
- **Expected Response:**
  - Status: 201
  - Body shape: `{ data: { slug: "new-event", ... } }`

#### API-EVENTS-010: Update event with admin token
- **Source:** `src/routes/events.ts`
- **Endpoint:** `PATCH /events/:slug`
- **Priority:** P1
- **Auth Required:** admin
- **Preconditions:**
  - Event with slug `existing-event` exists.
- **Request:**
  - Method: PATCH
  - Path: `/events/existing-event`
  - Headers: `Authorization: Bearer <admin-token>`
  - Body: `{ "title": "Updated Title" }`
- **Expected Response:**
  - Status: 200
  - Body shape: `{ data: { title: "Updated Title", ... } }`

## Edge Cases

| ID | Edge Case | Endpoint | Priority | Description |
|----|-----------|----------|----------|-------------|
| API-EVENTS-003 | GET draft event | GET /events/:slug | P0 | Draft events throw 404. |
| API-EVENTS-005 | Missing slug | GET /events/:slug | P0 | Non-existent slug → 404 |
| API-EVENTS-007 | Missing slots info | GET /events/:slug/slots | P0 | Non-existent slug → 404 |
| API-EVENTS-009 | Invalid Zod body | POST /events | P1 | Missing `slug` → 400 (Bad Request) |
| API-FAQS-003 | Missing Question | POST /faqs | P1 | Missing validation fails 400 |
| API-EDGE-AUTH-001 | Guest hits protected boundary | POST /events | P0 | No token on protected route → 401 |
| API-EDGE-AUTH-002 | User hits admin route | POST /events | P0 | User token on admin route → 403 |

*(Note: Guest hits to `/users/me` return 200 { data: null } by design).*
